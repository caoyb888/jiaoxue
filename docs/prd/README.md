## 单功能 PRD 模板（每个功能条目统一用此结构填写）

```markdown
### F{模块号}-{序号}　{功能名}

- **用户故事**：作为 <角色>，我希望 <动作>，以便 <价值>。
- **触达路径**：从 <入口页> → <路由/操作> 到达本功能。
- **角色与权限**：教师 / 学生 / 管理员分别可见可操作的范围。
- **关键交互 / 状态**：正常态 / 空态 / 加载态 / 错误态 / 无权限态。
- **三端适配预期**：PC(1280) / 平板(768) / 手机(375) / 小程序 各自表现。
- **关联约束**：涉及的 CLAUDE.md §8 红线（如 C1/C2/C6）或 API 规范条目。
- **当前实现状态**：✅/🟠/⚠️/❌（引用 00-差距盘点）+ 代码位置。

**验收标准（AC）**

| # | Given（前置） | When（操作） | Then（期望结果） | 测试结果 |
|---|---|---|---|---|
| 1 | | | | ⬜ 待测 |
| 2 | | | | ⬜ 待测 |
```
