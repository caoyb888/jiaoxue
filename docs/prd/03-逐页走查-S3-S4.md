## ⛳ S4 全模块根因（务必先读）

`frontend/packages/api/src/modules/exam.ts` 的 **`examApi`** 对象、`StudentAnswerPage` 内联的 **`studentExamApi`**、`DocxImportPanel` 内联的 **`docxApi`**，返回类型仍声明为 `{ code: number; data: T }` 信封，且 hooks/组件再去取 `.then(r => r.data)` 或 `res.data`。

但 `client.ts` 响应拦截器（第 28–39 行）**已把 `Result<T>` 信封解包为 `data`**。因此这些函数 resolve 出来的本就是业务数据，`r.data` / `res.data` 恒为 `undefined`。这与已知的 **`api-client-unwrap-pattern`** 完全同型。

> 对照：S5 的 `examStudentApi` / `examPublishApi`（exam.ts 第 300–342 行）已改为「直返业务数据 + 双泛型」写法，是正确范式。**S4 这一批从未被迁移**——因为 S4 页面虽挂了路由，但题库/试卷依赖真实数据，过去走查少、bug 一直潜伏。

**后果**：S4 题库、组卷、随堂答题、批量导入四块的**数据读写在运行态几乎全废**（列表恒空 / 加载报错 / 提交无反馈），见下表 🔴 项。TypeScript 不报错（`r` 类型是声明的信封，`.data` 合法），所以 typecheck 全绿也掩盖了它。

## 汇总：必修清单

### 🔴 功能性损坏（同一根因，建议一次性修复）

迁移 `exam.ts::examApi`、`StudentAnswerPage::studentExamApi`、`DocxImportPanel::docxApi` 到 S5 已用的「直返业务数据 + 双泛型」范式，并删除所有 `.then(r => r.data)` 与 `res.data`：

1. `useQuestionBanks` / `useQuestions` / `useExamPapers` / `usePaperQuestions` → 去掉 `.then(r => r.data ?? [])`，函数类型改为直返 `QuestionBankVO[]` / `PageResult<QuestionVO>` / `ExamPaperVO[]` / `PaperQuestionVO[]`
2. `QuestionBankPage:41` `res.data?.id` → `res.id`；`ExamPaperPage:69` 同理
3. `DocxImportPanel:64/72` `res.data` → `res`
4. `StudentAnswerPage:52` `activeRes?.data` → `activeRes`（注意 nullable）；`:73` `res.data ?? null` → `res ?? null`

> 影响面：题库、组卷、随堂答题、批量导入四大块运行态全部从「空/报错」恢复可用。**这是 S4 看似完成实则不可用的首因。**

### 🟡 功能缺口（对 PRD）

**已修复（2026-06-21，全栈）：**
- ✅ **弹幕 WebSocket 广播**：根因是全栈断链——`InteractionServiceImpl` 发的 Kafka 消息用裸 `Map{type:...}`，与 `TeachingEvent{eventType,payload}` 结构不符 → 反序列化后 `eventType=null` 被消费者丢弃；且 `TeachingEventConsumer` 只处理 `QUESTION_*`，无 `BARRAGE` 分支。修复：①interaction 改发规范 `TeachingEvent("BARRAGE",…)`；②notify 消费者补 `BARRAGE/ROLL_CALL/ATTEND_COUNT` 三分支 → `broadcastBarrage/RollCall/AttendCount`；③前端新增 `useLessonTopic` STOMP hook（`@stomp/stompjs`，连 `/ws/websocket`），`BarragePage` 订阅 `/topic/lesson/{id}/barrage` 统一上屏（去掉本地回显避免重复）
- ✅ **签到人数 WebSocket 实时推送**：`AttendanceServiceImpl.attend` 计数 +1 后发 `TeachingEvent("ATTEND_COUNT",{count})`；`AttendancePage` 用 `useLessonTopic` 订阅 `/topic/lesson/{id}/attend` → `setWsCount`（替换原空桩 useEffect）
- ✅ **RichTextView 经 DOMPurify**：`DOMPurify.sanitize(html)` 后再 `dangerouslySetInnerHTML`（§6.5）
- ✅ 顺带修复 **随机点名 WS 广播**后端断链（`ROLL_CALL` 同根因，消费者已补分支；前端 RollCallPage 仍用 mutation 同步结果，未加订阅）

> 验证：`pnpm -r typecheck` 7 包全绿；`mvn -pl edu-interaction test` 4/4 通过（已补 `KafkaTemplate` mock）；edu-notify 编译通过。**运行态待 /verify**（需内网机 `pnpm install` 装 `@stomp/stompjs`、重启 notify + interaction + vite）。
