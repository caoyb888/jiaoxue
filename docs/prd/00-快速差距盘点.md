> 目的：在补做 PRD 之前，先用一张表看清"技术方案应有功能 ↔ 前端实际现状"的差距，判断差距落在 **没做**、**做了但点不到**、还是 **做歪了**。
> 盘点日期：2026-06-20　|　范围：Sprint 1–5（模块1/3/5 + 课程课堂 + 登录鉴权）
> 盘点方式：以 `开发计划Sprint-Plan.md` 的 Story 反推应有功能，对照 `frontend/` 实际代码（路由表 / 页面 / api 层 / 关键 hook）。

## 一句话结论

**S1–S4 的前端基本可达；S5（在线考试与监考）整套页面已开发完成但成了"孤儿"——`ExamEnterPage`/`ExamAnswerPage`/`MonitorDashboardPage`/`ReviewPage` 四个页面均未挂载到 `router/index.tsx`，Dashboard 也无入口。** 考试是系统最核心、最显眼的功能，却在前端"点不进去"，这是你感觉"与技术方案差距大"的首要原因。其次是若干报告/统计类页面缺失，以及小程序端、大屏端覆盖不足。

## 差距归类小结（决定 PRD 详写优先级）

| 差距类型 | 典型条目 | 处理建议 |
|---|---|---|
| **做了但点不到**（最高优先，改动最小见效最大） | S5 全部 4 页（进入/答题/阅卷/监考） | 接路由 + Dashboard 入口 + 厘清与 StudentAnswerPage 边界 → 立刻可用，先写这几页 PRD+AC |
| **没做** | 课堂积分、课堂报告页、考试发布配置页(存疑) | 在 PRD 中补全用户故事与 AC，排期返工 |
| **覆盖不足** | 小程序端功能少、bigscreen 空壳 | 小程序按 PRD 明确"必须覆盖哪些核心场景"；大屏属 S7 |
| **可能做歪**（需运行态走查） | 三端断点、双视角、弹幕/点名样式、JWT 存储合规 | PRD 写清 AC 后逐页验收 |
