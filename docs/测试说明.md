## 1. 后端单元 / 集成测试（Maven）

- 框架：JUnit 5 + Mockito（单测），`@SpringBootTest` / MockMvc（集成）。
- 位置：各服务 `src/test/java/...`（镜像 main 目录结构），命名 `{Class}Test.java`。
- 覆盖率目标：Service 层 ≥ 70%（见 CLAUDE.md §10）。

```bash
# 单个服务
cd backend && mvn test -pl edu-exam
# 全部
cd backend && mvn test
```

必测场景（CLAUDE.md §10.2）：签到幂等、交卷幂等、AI Prompt 安全层过滤、教务 ETL 增量同步。

## 5. 运行前置与注意事项（重要）

### 为什么 E2E 必须在内网机运行
系统运行在内网机（`100.84.68.115` / `onlyserver`），E2E 测试需：
- 经 `docker exec edu-redis` 读取 **dev 短信验证码**（`sms:code:{phone}`）完成登录；
- 经 `docker exec edu-mysql` **核对落库**（如 attendance / student_answer）。

这些依赖宿主机上的容器，**本机（开发机）跑不了**。

### dev 短信登录
测试账号用手机号 + 验证码登录（**不点"获取验证码"也可**，dev 码可重置为 123456/24h）：

| 角色 | 手机号 | 备注 |
|------|--------|------|
| 管理员 admin | `13800000001` | — |
| 教师 teacher01 | `13800000002` | — |
| 学生 student01 | `13800000011` | `sys_user.id = 3` |

```bash
# 重置全部测试账号验证码为 123456（dev）
ssh onlyserver 'bash ~/smart-edu/infra/scripts/reset-sms-codes.sh'
```

### 串行执行（workers=1）
`e2e/playwright.config.ts` 已设 `workers: 1`。**勿改回并行**：全套共用同一后端 + 同一批 dev 手机号，并行会**抢短信验证码**导致登录竞态与共享状态时序 flaky。

### 测试数据保留
E2E 每次运行会在系统中**创建真实数据**（课堂/题库/试卷/发布/签到/交卷等）并**保留**（按约定不清理）。用例设计为每次创建独立数据、互不干扰、可重复运行。
